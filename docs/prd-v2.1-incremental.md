## 2. 产品定义

### 2.1 产品目标

1. **体验止血（v2.1.0）**: 消除代码重复与竞态缺陷，降低用户感知的卡顿和闪烁，2 周内交付可感知的品质提升。
2. **功能升级（v2.1.0）**: 构建记忆曲线可视化、自动混合学习模式、高质量离线发音三大差异化能力，确立"科学背词"心智。
3. **架构演进（v2.2.0）**: 完成前端模块化拆分、数据层性能优化、API 规范化，为后续多端扩展铺路。

#### P0-03：修复 `confirmResolver` 全局单例竞态

| 维度 | 内容 |
|------|------|
| **当前问题** | 应用使用全局单例 `confirmResolver` 管理确认弹窗。当弹窗 A 触发后、用户尚未响应时弹窗 B 也触发，B 会覆盖 A 的 resolver，导致 A 的 Promise 永不 resolve（界面卡死）。 |
| **目标** | 改为队列化（queue）处理：同一时刻只有一个活跃的确认弹窗，后续请求排队；或改为栈式（stack）：新弹窗覆盖旧弹窗并 reject 旧 Promise。 |
| **涉及文件** | `src/app.js` |
| **验收标准** | 1. 连续快速触发 3 次确认弹窗，每个弹窗都能正常交互并 resolve/reject。2. 不存在 Promise 悬挂（无未处理的 rejection）。3. E2E 测试新增竞态场景。 |
| **依赖** | 无 |

#### P2-15：事件监听器生命周期管理

| 维度 | 内容 |
|------|------|
| **当前问题** | 事件监听器通过 `addEventListener` 分散绑定，缺少统一的注册表。模式切换、弹窗打开/关闭时可能残留未解绑的监听器，造成内存泄漏和重复触发。 |
| **目标** | 实现 `EventListenerRegistry`：提供 `register(element, event, handler, scope)` 和 `disposeScope(scope)`。每个 UI 模块（弹窗、模式、面板）使用独立 scope，模块销毁时批量解绑。 |
| **涉及文件** | 新建 `src/event-registry.js`、修改所有涉及事件绑定的模块 |

## 4. UI/UX 设计要点

### 4.1 记忆曲线面板（P1-07）

```
┌──────────────────────────────────────────────┐
│  📊 我的记忆曲线          [7天] [14天] [30天] │
├──────────────────────────────────────────────┤
│                                              │
│   100% ┊                                  理想│
│    80% ┊  ╲                              ···│
│    60% ┊    ╲          个人                 │
│    40% ┊      ╲      ╱╲                    │
│    20% ┊        ╲  ╱    ╲                  │
│     0% ┊──────────╱────────╲─────────────   │
│        └──────────────────────────────────   │
│         D1   D3   D7   D14  D30            │
│                                              │
│  说明：实线为你的记忆保持率，虚线为理论遗忘曲线│
└──────────────────────────────────────────────┘
```

**设计要点**:
- Canvas 绘制，圆角折线图，数据点带小圆点标记
- 理想曲线使用半透明虚线（`rgba(47,111,115,0.4)`）
- 个人曲线使用主色实线（`#2f6f73`），线宽 2px
- Y 轴范围 0%-100%，步长 20%
- X 轴标注关键复习节点（D1/D3/D7/D14/D30）
- 顶部时间段切换使用 Segmented Control 风格
- 深色模式下背景 `#1a1a2e`，个人曲线亮度提升

### 4.2 单词卡片过渡动画（P0-05）

**前进（下一词）动画**:
```
旧卡片: opacity 1→0, translateX 0→-20px, 200ms ease-out
新卡片: opacity 0→1, translateX 20px→0, 200ms ease-out
```

**后退（上一词）动画**:
```
旧卡片: opacity 1→0, translateX 0→20px, 200ms ease-out
新卡片: opacity 0→1, translateX -20px→0, 200ms ease-out
```

**CSS 实现要点**:
```css
.word-card {
  transition: opacity 0.2s ease-out, transform 0.2s ease-out;
}
.word-card.slide-forward-out {
  opacity: 0;
  transform: translateX(-20px);
}
.word-card.slide-backward-out {
  opacity: 0;
  transform: translateX(20px);
}
```

- 动画使用 CSS transition（非 JS animation），GPU 加速
- 动画时长 200ms，不阻塞键盘事件
- 动画期间 `pointer-events: none` 防止误触

### 4.3 智能混合模式 UI（P1-08）

```
┌──────────────────────────────────────────────┐
│  [🧠 混合] [📖 复习] [🆕 新学] [💪 强化] [📝 测试] │
└──────────────────────────────────────────────┘
```

**设计要点**:
- 混合模式为默认选中项，使用 Segmented Tabs 风格
- 选择非默认模式后显示"切换回混合"提示入口
- 混合模式下进度条分段显示：`████░░░░░░ 复习 6/10 · 新学 4/20`
- 当前为复习词时，卡片右上角显示 "复习" 标签（`<Badge>`）；新学词显示 "新学" 标签
- 标签颜色：复习 = `#2f6f73`（主色），新学 = `#b7791f`（暖色）
